[07] 객체지향 프로그램의 본질

	1. 메소드 프로그램의 한계
		메소드를 이용해 로직을 재사용할 수 있게 되었고, 개발이 비교적 효율적이어졌지만,
		문서 내에 메소드 수가 너무 많아지면 유지보수가 어려워짐.

	2. 객체의 개념, 클래스의 이해
		객체는 '같은 종류의 데이터와 메소드가 함께 있는 구성체'
		객체는 메소드와 비슷하지만, 그 상위 개념. 완전한 독립체.
		클래스는 객체를 만들기 위한 설계도이다.
		(cf) 객체 모델링: 현실세계나 추상적인 내용의 속성과 동작을 추려내어 소프트웨어 객체의 필드와 메서드로 정의해 나가는 과정

	3. 객체 자료형과 변수와의 관계
		기초 데이터 변수의 경우, 무엇이든 담을 수 있는 주머니를 만들어놓고 주머니에 값을 직접 넣는다.

	4. 객체 자료형과 데이터와의 관계
		객체는 클래스로부터 생성
		생성된 객체는 동일한 클래스에서 생성되었다 해도 독립체.
		객체 안에 종속된 데이터도 완전히 별개의 데이터가 된다.
