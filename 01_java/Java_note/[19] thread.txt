[19] 스레드

	1. 프로세스와 스레드
		멀티 프로세스는 여러 프로그램을 돌리는 개념.
		멀티 스레드는 같은 프로그램 안에서 여러 작업을 동시에 하는 개념(예를 들어 카카오톡)
		하나의 새로운 프로세스를 생성하는 멀티프로세스보다 새로운 쓰레드를 하나 생성하는 것이더 적은 비용이 듬.
		스레드는 N개의 스레드가 객체 하나를 공유하는 방식 혹은 n개의 스레드가 각자 객체 N개를 사용하는 방식을 사용한다.

		1-1. 멀티 스레드의 장점
		-자원을 보다 효율적으로 사용할 수 있다.
		-사용자에 대한 응답성이 향상된다.
		-작업이 분리되어 코드가 간결해진다.

		1-2. 멀티 스레드의 단점(멀티 프로세스의 단점이기도 함)
		-동기화에 주의할 것
		-데드록 발생에 주의할 것
		-위와 같이 고려할 것이 많음

	2. 스레드의 생성주기
		1) 생성
		2) 실행-실행가능상태(준비상태)
		3) 대기
		4) 완료
		
	3. 멀티 스레드 문법
		1) Runable 인터페이스 구현을 통한 Target과 Thread
		2) Thread 클래스 상속을 통한 Thread

	4. Synchronized(동기화)
		1) 객체에 선언된 인스턴스 변수를 스레드에서 공유하면 인스턴스의 값에 영향을 미친다.
		Synchronized는 이런 경우를 대비해 사용되는 예약어. 먼저 수행되는 스레드의 모든 작업이 끝날때까지 다른 스레드는 기다리도록 하는 예약어이다.
		즉, 하나의 자원을 여러 태스크가 동시에 사용하려고 할 때, 한 시점에서 하나의 태스크만이 사용할 수 있도록 하는 것.
