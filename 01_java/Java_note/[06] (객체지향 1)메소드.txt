[06] (객체지향 1)메소드

	1. 객체지향 이전: 절차지향 방식
		위에서부터 순서대로 진행. 코드의 유지보수가 어렵다.
		절차지향 방식의 비효율적인 방법 개선을 위해 등장한 것이 메소드.
		로직만 만들어놓고, 데이터를 주면 메소드가 알아서 결과값을 반환하는 방식으로 점차 발전

	2. 객체지향 등장 배경과 이해
		메소드를 이용한 방식은 메소드가 증가함에 따라 코드 양이 많아지고 어려워짐
		그래서 객체지향 프로그래밍이 등장했다.
		객체란, 동일 성질의 데이터와 메소드를 한 곳에 모아두고 필요한 곳에서 언제든지 이용할 수 있게 만든 덩어리.

	3. 메소드?
		작업 수행을 위한 명령문의 집합.
		특정 값을 입력받아 처리하고, 결과를 반환함.
		반복적으로 수행되는 여러 문장을 메소드로 작성한다.
